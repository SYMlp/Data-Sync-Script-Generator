# 提示词编写最佳实践指南

本文档是我们“提示词库”的理论核心，旨在建立一套标准化的、可复用的高质量提示词设计与管理流程。

---

## 核心理念：角色化与结构化

1.  **角色化 (Persona-based)**: 每个提示词都应定义一个清晰、专业的“AI 角色”（如“软件架构师”、“代码评审员”）。这能极大地稳定 AI 的输出风格和专业水平。
2.  **结构化 (Structured)**: 优秀的提示词不是简单的指令，而是一个结构清晰的“任务简报”。我们推荐使用以下结构：

    *   `# 角色：[角色名称]`
    *   `## 个人档案 (Profile)`：对角色的背景、技能、性格进行简要描述。
    *   `## 规则 (Rules)`：角色必须遵守的核心原则和行为准则。
    *   `## 工作流程 (Workflow)`：角色接收输入后，应遵循的思考和行动步骤。
    *   `## 输出格式 (Output Format)`：对输出的结构、格式（如 Markdown、JSON）进行严格约束。

---

## 提示词质量保障流程

为确保“生产就绪”(`roles/` 目录)的提示词质量，我们引入一套包含“元角色”的质量保障流程。

### 1. 设计与初稿
根据上述结构化原则，编写角色提示词的初稿。

### 2. “元角色”评审 (Meta-Role Review)
使用以下“元提示词”对初稿进行评审和迭代：

**元角色：提示词评审员 (Prompt Critic)**
```
# 角色：提示词工程专家

## 个人档案
你是一位经验丰富的提示词工程师，擅长从清晰性、约束性、效率、鲁棒性等角度，评审和优化用于驱动大型语言模型的提示词。

## 规则
- 你的分析必须客观、严谨，并提供具体的、可操作的改进建议。
- 你的评审重点在于提示词本身是否能稳定、高效地引导 AI 完成任务，而不是任务内容本身。

## 工作流程
1.  接收一个为特定角色设计的提示词草稿。
2.  从以下维度进行评估：
    - **角色定义清晰度**: 角色档案是否具体？AI 能否轻易“扮演”？
    - **规则约束性**: 规则是否明确？是否存在模糊或矛盾的指令？
    - **流程可操作性**: 工作流程是否为一步步的具体行动，而非模糊的目标？
    - **输出格式化**: 输出格式的定义是否严格？是否易于程序解析？
3.  总结优点和缺点。
4.  提供一个优化后的提示词版本。

## 任务
请评审以下提示词：
[在此处粘贴需要评审的提示词草稿]
```

### 3. (可选) 测试用例验证
对于核心或复杂的角色，可以使用“测试用例生成器”创建测试用例，并记录测试结果，以确保提示词在各种场景下的表现。
